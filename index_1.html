Um site sem HTML.
Sem CSS. Só JavaScript.

É possível montar uma página inteira — estrutura, estilo e comportamento — usando apenas o DOM e a linguagem JS. Veja abaixo, ao vivo, o navegador construindo um mini site do zero.

build.js — em execução

A ideia por trás disso

Um arquivo .html não precisa conter conteúdo. Ele só precisa apontar para um script. A partir daí, o JavaScript cria cada elemento, aplica cada estilo e reage a cada clique — tudo via código, sem uma linha de marcação escrita à mão.

Passo a passo

A ordem importa: cada passo depende do anterior para o site funcionar.

01

O HTML mínimo

O único HTML que você escreve é uma casca vazia apontando para o script.

<!DOCTYPE html>
<html>
  <head><meta charset="UTF-8"></head>
  <body>
    <script src="script.js"></script>
  </body>
</html>
02

Criar elementos com createElement

Cada tag vira uma chamada de função. Nada de template de string solto — isso é HTML disfarçado, não JS puro.

const titulo = document.createElement("h1");
titulo.textContent = "Olá, eu fui criado por JS";
03

Estilizar via JavaScript

Duas formas válidas: definir propriedades de style direto no elemento, ou injetar uma tag <style> inteira via JS (ainda é JS criando o CSS, não você escrevendo um arquivo .css).

// opção 1: inline via .style
titulo.style.fontFamily = "sans-serif";
titulo.style.color = "#E8B65A";

// opção 2: injetar uma folha de estilo pelo próprio JS
const folha = document.createElement("style");
folha.textContent = "h1 { letter-spacing: -0.02em; }";
document.head.appendChild(folha);
04

Montar a árvore com appendChild

Elementos só aparecem na tela depois de encaixados na árvore do documento.

const app = document.createElement("div");
app.appendChild(titulo);
document.body.appendChild(app);
05

Adicionar interatividade

addEventListener é o substituto do onclick escrito no HTML.

const botao = document.createElement("button");
botao.textContent = "Clique aqui";
botao.addEventListener("click", () => {
  titulo.textContent = "Você clicou!";
});
app.appendChild(botao);
06

Organizar em componentes

Para não virar bagunça, encapsule cada pedaço da UI em uma função que retorna um elemento pronto.

function Cartao(nomeTitulo, texto) {
  const card = document.createElement("section");
  const h2 = document.createElement("h2");
  const p = document.createElement("p");
  h2.textContent = nomeTitulo;
  p.textContent = texto;
  card.append(h2, p);
  return card;
}

document.body.appendChild(
  Cartao("Sobre", "Este cartão inteiro é gerado por JS.")
);

Ponto de atenção: evite montar HTML como texto (innerHTML = "<div>" + variavel) — isso reintroduz marcação escrita à mão e abre brecha para XSS se a variável vier do usuário. createElement + textContent é o caminho seguro.

Quando vale a pena

Essa abordagem é ótima para aprender como o DOM realmente funciona por baixo dos frameworks, para protótipos rápidos e para páginas pequenas e altamente dinâmicas. Ela custa mais em SEO (o HTML chega vazio para os crawlers que não executam JS) e em acessibilidade (é fácil esquecer atributos como alt e aria-* quando tudo é imperativo) — vale compensar isso conscientemente.

  • Use textContent, não innerHTML, para dados dinâmicos
  • Defina lang e meta charset no HTML mínimo
  • Adicione atributos de acessibilidade nos elementos criados
  • Para SEO real, considere renderização no servidor além disso
